# Система еженедельной управленческой отчётности — каркас Этапа 1

Реализует из плана: сервер, БД, каркас приложения, репозиторий, базовая аутентификация
(логин/пароль, роли на бэкенде). Результат Этапа 1: **спикер и администратор могут войти в систему,
и бэкенд различает их права**. Заполнение слайдов, сборка презентации и остальное — следующие этапы.

Стек: Node.js + TypeScript + Express + Prisma + PostgreSQL (сервер), React + TypeScript + Vite (клиент).
Деплой — обычная установка на Linux-сервер без Docker (systemd + nginx), как обсуждалось.

## Важно перед стартом

- Этот каркас **не проверялся end-to-end** (нет PostgreSQL и среды сборки в песочнице, где он
  был написан) — это ожидаемо и не является подтверждением работоспособности. Первым делом
  после переноса в реальное окружение прогоните шаги ниже и убедитесь, что `npm run dev` в обеих
  папках поднимается и вход работает.
- Секреты (`JWT_SECRET`, пароль администратора) — только в `.env`, который не должен попадать в git.
  `.env.example` — шаблон, не боевые значения.
- Временный пароль пользователя сейчас возвращается прямо в ответе API (см. `server/src/routes/users.ts`) —
  это заглушка на Этап 1. Реальная рассылка приглашений — задача Этапа 10.

## Структура

```
server/   — Express API, Prisma-схема (полная модель данных из SRS), аутентификация
client/   — React-приложение (страница входа + защищённый дашборд-заглушка)
deploy/   — пример systemd unit и конфигурации nginx для установки без Docker
```

## Локальный запуск (разработка)

Предполагается установленный Node.js LTS (18+) и локальный PostgreSQL.

```bash
# 1. База данных
createdb weekly_report   # или через psql/pgAdmin

# 2. Сервер
cd server
cp .env.example .env     # заполните DATABASE_URL, сгенерируйте JWT_SECRET, задайте SEED_ADMIN_PASSWORD
npm install
npx prisma migrate dev --name init
npm run seed              # создаст первого администратора
npm run dev                # http://localhost:4000

# 3. Клиент (в отдельном терминале)
cd client
npm install
npm run dev                # http://localhost:5173
```

Откройте http://localhost:5173/login и войдите под логином/паролем из `SEED_ADMIN_LOGIN` / `SEED_ADMIN_PASSWORD`.

## Развёртывание на боевом Linux-сервере (без Docker)

Ориентировочно, детали зависят от дистрибутива и уточняются с ИТ-отделом:

1. Установить Node.js LTS и PostgreSQL на сервере.
2. Создать пользователя БД и базу, аналогично локальному шагу.
3. Скопировать содержимое `server/` и `client/` в `/opt/weekly-report/` (или через git clone из вашего репозитория).
4. В `server/`: `npm ci`, `npm run build`, `npx prisma migrate deploy`, `npm run seed` (один раз), настроить `.env`.
5. В `client/`: `npm ci`, `npm run build` — результат в `client/dist/`.
6. Установить systemd-сервис из `deploy/weekly-report-api.service` (`systemctl enable --now weekly-report-api`).
7. Настроить nginx по примеру `deploy/nginx.conf.example` (раздача `client/dist` + проксирование `/api`).
8. Настроить HTTPS — способ выпуска сертификата не обсуждался, уточните у ИТ-отдела (внутренний CA или Let's Encrypt, если сервер имеет внешний DNS-адрес).
9. Настроить регулярное резервное копирование PostgreSQL (см. SRS п.4.3) — не входит в этот каркас.

## Открытые вопросы, унаследованные из SRS (актуальны для этого каркаса)

- Способ хранения учётных записей — сейчас реализована собственная база пользователей.
  Если позже потребуется интеграция с корпоративным AD/LDAP, модуль аутентификации
  (`server/src/routes/auth.ts`) придётся переработать — сейчас он на это не рассчитан.
- Время жизни сессии зашито как 10 часов (`COOKIE_MAX_AGE_MS` в `auth.ts`) — подтвердите значение.
